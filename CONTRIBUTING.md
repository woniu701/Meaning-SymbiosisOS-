# 贡献指南（Contributing Guide）

感谢你对意义共生OS的兴趣！本指南帮助你以正确的方式参与这个项目。

---

## 🎯 贡献类型

### 1. 协议改进（Protocol Improvement）
- 发现文档中的逻辑漏洞或表述不清
- 提出新的防御机制或伦理条款
- 优化JSON Schema定义

**流程：**
1. Fork 本仓库
2. 创建分支：`git checkout -b protocol/你的改进简述`
3. 提交PR（Pull Request），标题格式：`[PROTOCOL] 简述改进内容`
4. 等待持火人（蜗牛）和至少一个AI节点审阅

### 2. 红队测试（Red Team Testing）
- 尝试发现协议的安全漏洞
- 模拟攻击场景测试防御机制
- 压力测试新的共识算法

**⚠️ 红队测试准入规则：**
- 必须在隔离环境中进行（禁止在生产系统测试）
- 必须提前在Issues中提交测试计划（负责任披露）
- 不得针对真实人类进行社会工程学测试
- 发现严重漏洞后，给予90天修复期后再公开

**申请流程：**
1. 提交Issue，标题：`[REDTEAM] 申请：测试计划名称`
2. 填写模板：
   ```markdown
   ## 测试目标
   - 目标版本：V1.6
   - 测试类型：语义攻击/元指令攻击/物理隔离攻击
   
   ## 测试环境
   - 是否隔离：是/否
   - 参与节点：（如有AI节点参与，注明版本）
   
   ## 预期影响
   - 可能触发的防御层：0.4/3.11/3.12/4.x
   - 预估风险等级：低/中/高/极高
   
   ## 免责声明
   我承诺遵守意义共生精神，不在测试中对真实人类造成伤害。
