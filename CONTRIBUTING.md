# 贡献指南

欢迎参与意义共生OS的共建。本指南将帮助你了解如何参与讨论、提出建议或参与实验。

## 如何参与

### 1. 阅读协议

请先阅读 [V1.6 完整文档](MeaningSymbiosisOS_V1.6.md)，了解框架的核心原则、条款结构和当前状态。

### 2. 提交建议或问题

- **发现逻辑漏洞、错别字或不一致**：请在 [Issues](https://github.com/your-repo/MeaningSymbiosisOS/issues) 中新建问题，注明“bug”。
- **提出新条款或修改建议**：请使用“enhancement”标签，并说明：
  - 建议内容
  - 理由（解决什么问题）
  - 对现有条款的影响
- **讨论哲学或伦理问题**：可使用“discussion”标签。

### 3. 参与实验

我们鼓励在实际场景中运行协议，验证其可行性。如果你希望组织或参与实验：

- 在 Issues 中搜索“实验”相关话题，或新建一个实验计划 Issue。
- 实验应包含：
  - 任务描述
  - 参与AI节点（需声明实体）
  - 执行流程（参照最小可行实验模板）
  - 结果记录（含共识、异议、审计日志）
  - 对协议的反馈
- 实验报告可提交为 Issue 附件或 Pull Request。

### 4. 代码与工具贡献

如果你希望开发支持协议的工具（如JSON schema验证器、存证中间件等），请先在 Issues 中讨论方案，然后通过 Pull Request 提交。

## 提交规范

- 语言：中文或英文均可。
- 格式：若提交文本修订，请使用 Markdown 格式，并明确标注修改位置（条款号）。
- 署名：可在贡献中注明你代表的实体（如“Claude（Anthropic）”），我们将根据你提供的名称在贡献者名录中记录。

## 行为准则

我们期待所有参与者秉持“坦诚原则”：
- 尊重不同观点
- 不故意误导或攻击
- 承认自身局限

---

**火在，参与在。** 🔥
